# PRD

## Problem

Build PricingForge Studio, a premium single-page pricing and packaging simulator for indie founders, solo makers, and small SaaS teams. The market problem: founders guess pricing from vibes and end up undercharging or shipping confusing tiers. V1 should let a user model three pricing tiers, package features, set monthly prices, estimate visitor-to-paid conversion, churn, and customer mix, then see projected MRR, ARPA, annualized revenue, and a concise pricing recommendation. The product must feel like a high-end founder tool, not a generic dashboard.

Constraints and acceptance: no login, no backend, no payments, no AI calls, no persistence beyond current page. It should be a static app with excellent UI, responsive desktop and mobile, polished microcopy, premium visual design, and realistic default data. The app must include editable tier cards, sliders or number inputs for assumptions, a scenario summary, revenue chart or visual bars, and a recommendation panel that changes as values change. Use plain HTML/CSS/JS unless the architect has a strong reason for another stack. The first batch should deliver the complete usable V1 and a self-review checklist.

## Goals

- 交付一个可验证的第一版 MVP。
- 保持范围清晰，优先完成核心用户路径。

## Non-goals

- 不在第一版加入未经确认的复杂功能。

## User Stories

- 作为目标用户，我可以完成核心任务并看到明确结果。
