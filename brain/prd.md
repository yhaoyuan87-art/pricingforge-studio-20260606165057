# PRD

## Problem

独立开发者和小型 SaaS 团队经常凭感觉定价：套餐边界不清、价格梯度太弱、主推档不明显，最终导致低 ARPA、高流失或用户不知道该买哪一档。PricingForge Studio 要把定价假设变成一个直观的中文决策台，让用户在上线前快速判断“这个价格结构是否靠谱”。

## Goals

- 让用户一眼知道产品用途：3 分钟判断 SaaS 定价是否靠谱。
- 提供清晰使用路径：输入增长假设、调整三档套餐、读取诊断建议。
- 支持三档套餐、客户分布、月访问量、付费转化率、月流失率的实时编辑。
- 实时输出 MRR、ARPA、年化收入、流失调整后收入、收入结构和中文建议。
- 提供可复制的汇报摘要，方便用户带到团队讨论。
- UI 必须达到可商用中文 SaaS 工具质感：高级、可信、直观、响应式。

## Non-goals

- 不做登录、云同步、支付、后端、AI 调用或团队协作。
- 不做复杂财务建模、税务、企业销售漏斗或多币种管理。
- 不保存用户输入到浏览器之外的任何地方。

## User Stories

- 作为独立开发者，我可以输入当前产品的流量、转化和流失假设，立即看到收入模型。
- 作为 SaaS 创业者，我可以调整三档套餐价格和卖点，看哪个套餐承担主要收入。
- 作为产品负责人，我可以复制一段中文摘要，用于团队定价 review。
- 作为首次访问用户，我不用读文档也能知道先做什么、后看什么。

## Acceptance Criteria

- 打开首页即可看到中文价值主张、使用步骤和“开始模拟”入口。
- 所有关键 UI 文案为中文。
- 修改任一核心输入后，指标、收入条和建议面板实时更新。
- 页面在桌面和移动端都无明显重叠、溢出或不可操作控件。
- `node tests/revenue-math.test.js` 通过。
